# 课堂竞技玩法设计文档 v1

## 1. 文档目标

本文档用于记录当前系统中“课堂竞技玩法”的设计方向、已落地能力、推荐扩展顺序与后续约束，作为后续继续完善玩法设计的基线。

本文档只讨论玩法形态，不重复展开任务系统、学习包、课堂模式的基础设计。

---

## 2. 核心原则

### 2.1 竞技是任务形态，不是独立系统

竞技玩法不单独建设一套题库、判题和内容系统，而是复用现有：

1. 任务组
2. 题型配置
3. 判题能力
4. 排名与结果展示
5. 教师端课堂模式

因此，竞技的本质是：

**同一批任务，以更强互动性、更高参与感的方式运行。**

### 2.2 优先做轻量化玩法

当前阶段不做：

1. 独立游戏大厅
2. 重动画型大屏游戏
3. 复杂多人乱斗
4. 课后自由房间系统

当前优先做：

1. 双人 PK
2. 全班挑战
3. 单题抢答
4. 擂台赛
5. 小组对抗

### 2.3 玩法必须服务教学

每一种玩法都必须满足至少一个教学目标：

1. 提高课堂参与率
2. 提高学生作答速度
3. 提高知识点检查效率
4. 提高课堂气氛与注意力
5. 将竞技结果回流到教师教学判断

---

## 3. 当前已落地玩法

### 3.1 全班挑战

当前状态：`已实现`

形态：

1. 教师在课堂模式中选择任务组
2. 发起 `class_challenge`
3. 全班学生参与作答
4. 教师端展示排行榜
5. 学生端展示个人结果页

当前已支持：

1. 排行榜
2. 排名、答对数、进度、用时
3. 教师结束挑战
4. 学生晚进入恢复挑战状态
5. 重复提交与结束态回放保护
6. 开场倒计时
7. 前三名突出展示
8. 结束课堂总结卡

### 3.2 双人 PK

当前状态：`已实现增强版`

形态：

1. 教师从课堂内学生中选择两名学生
2. 发起 `duel`
3. 两名学生进入同一场 PK
4. 教师端展示实时比分
5. 当前已补大屏展示态

当前已支持：

1. 双人选择
2. 排行与用时计算
3. 结束态展示
4. 教师端投屏展示
5. 重连与重复提交保护
6. 逐题比分变化
7. 当前领先者 / 平局展示
8. 教师提前结束的结果回放

### 3.3 单题双人抢答

当前状态：`已实现`

形态：

1. 教师从任务组中选择 1 道题
2. 选择 2 名学生
3. 发起 `single_question_duel`
4. 两名学生围绕同一题抢答
5. 教师端和投屏态显示本题胜负结果

当前已支持：

1. 仅允许单选、判断、图片理解、纠错题进入抢答
2. 先答对者立即胜出
3. 先答错者本题失败，等待对方继续作答
4. 超时无人答对时判定平局
5. 非参与学生只能围观
6. 抢答结果回流到课堂历史与现有分析体系

## 4. 当前允许接入竞技的任务类型

当前已允许接入课堂竞技：

1. `single_choice`
2. `multiple_choice`
3. `fill_blank`
4. `true_false`
5. `matching`
6. `sorting`
7. `image_understanding`

其中 `single_question_duel` 仅允许：

1. `single_choice`
2. `true_false`
3. `image_understanding`
4. `error_correction`

当前不接入：

1. `reading`
2. 开放题
3. 口语题
4. 复杂表达输出题

原因：

1. 判定不够即时
2. 用时和公平性不好统一
3. 不适合当前轻量竞技目标

## 5. 推荐扩展玩法

以下玩法按推荐优先级排序。

### 5.1 单题双人抢答

推荐等级：`已落地，继续优化`

玩法说明：

1. 教师发布 1 道题
2. 两名学生同时看到题目
3. 先提交且答对者得分
4. 先提交但答错可按规则失分或本题失效

适合题型：

1. 单选
2. 判断
3. 图片理解
4. 纠错题

推荐原因：

1. 开发成本低
2. 课堂节奏强
3. 投屏表现好
4. 适合高频使用

### 5.2 多题双人 PK 增强版

推荐等级：`高`

可增强方向：

1. 回合制
2. 三局两胜
3. 每题即时比分波动
4. 中场提示与结束态强化

适合题型：

1. 单选
2. 配对
3. 排序
4. 图片理解

### 5.3 全班挑战增强版

推荐等级：`高`

可增强方向：

1. 开场倒计时
2. 更强实时排行榜
3. 前三名突出展示
4. 课堂总结页

适合场景：

1. 课前热身
2. 课中检查
3. 课尾总结

### 5.4 擂台赛

推荐等级：`中高`

玩法说明：

1. 一名学生作为擂主
2. 新挑战者上场
3. 胜者留场
4. 可连续守擂

适合题型：

1. 单题抢答
2. 判断
3. 图片理解
4. 情境选择

推荐原因：

1. 课堂戏剧性强
2. 观战体验好
3. 适合直播课堂

### 5.5 小组对抗

推荐等级：`中`

玩法说明：

1. 班级分为两组或多组
2. 各组累计总分
3. 可按轮次或总答题量计分

适合：

1. 班级规模较大时提高参与面
2. 避免只聚焦两个学生

### 5.6 闯关模式

推荐等级：`中`

玩法说明：

1. 连续 3-5 题为一关
2. 达标后进入下一关
3. 可用于课堂或课后轻挑战

### 5.7 淘汰赛 / 生存赛

推荐等级：`低到中`

玩法说明：

1. 一轮一题
2. 答错暂时淘汰
3. 最后留下者获胜

注意：

1. 课堂气氛强
2. 但容易让部分学生快速失去参与
3. 不建议作为高频主玩法

## 7. 玩法设计约束

后续新增玩法时，必须遵守以下约束：

1. 不新增独立题库系统
2. 必须复用现有任务组与题型注册表
3. 必须复用现有判题能力或完成状态能力
4. 必须定义排行榜规则
5. 必须定义异常状态处理：
   - 断线
   - 重连
   - 重复提交
   - 教师提前结束
6. 必须定义结果回流方式：
   - 是否进入课堂分析
   - 是否进入学习记录
   - 是否影响后续推荐
