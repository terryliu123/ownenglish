# 互动课堂执行清单

## 1. 白板 / 互动管理 / 学生端运行态

### P0-1 白板运行态单一真相源
- 去掉白板模式对任务/挑战的乐观本地真相
- 只认房间状态和 WebSocket 事件
- 运行态与列表展示态彻底分离

### P0-2 学生端参与态派生化
- 移除页面本地 `isChallengeParticipant`
- 只从 challenge 对象派生参与/围观状态

### P0-3 挑战结束态统一
- 统一教师端、白板、学生端的结束态
- 结束后不允许旧状态残留

## 2. 实时反馈

### P0-4 提交后即时反馈
- 提交立即进入等待结果态
- 提交失败有明确提示
- 不再出现“点击后无反馈”

### P0-5 排行榜与进度一致性
- 排行榜只认服务端 score data
- 不再由前端推导第二套排名

### P0-6 白板显示稳定
- 学生进入课堂后立即显示在白板
- 学生提交后白板提交数实时变化
- 分享/举手状态不再卡死

## 3. 白板与互动管理职责收口

### P1-1 白板模式职责固定
- 白板主入口只保留讲解与发起互动
- 互动进行中的展示留在白板
- 不再继续承载复杂历史和分析

### P1-2 课堂模式更名为互动管理
- 文案层统一改成“互动管理”
- 入口职责改成任务、历史、分析
- 和白板模式不再争主入口

### P1-3 发起动作统一
- 白板和互动管理都使用同一套动作命名
- 发布 / PK / 抢答 / 全班挑战入口一致
- 校验提示统一

## 4. 结果与结束

### P1-4 结束动作统一
- 结束普通任务
- 结束 PK
- 结束抢答
- 结束全班挑战

结束后统一刷新：
- 待发布
- 已完成
- 历史任务
- 当前白板状态

### P1-5 结果摘要统一
- 当前胜者
- 平局
- 参与人数
- 进度
- 用时

## 5. 老师体验

### P1-6 白板中继续讲解
- 互动结束后快速回到白板讲解
- 投屏关闭与白板恢复动作清晰

### P1-7 控场信息强化
- 谁已提交
- 谁未提交
- 当前领先者
- 当前第 N 名

## 6. 回归任务

### P0 回归集
- 普通任务发布
- 白板发题
- 双人 PK
- 单题抢答
- 全班挑战
- 晚加入
- 断线重连
- 教师提前结束

### P1 回归集
- 白板与互动管理路径一致
- 历史任务按场次正确查看
- 状态刷新不需要手动操作

## 7. 当前不进入

- 大屏过程型互动实现
- 新题型扩展
- 学习包回归
- 课后约战
- 平台化后台能力扩张
## 8. 上线前补充任务

### P1-8 登录后首页 / 引导首页
- 登录后增加一个独立首页，用于展示系统的操作流程和使用范式
- 目标：
  - 简单易懂
  - 视觉美观
  - 让老师首次登录后知道“先做什么、从哪里开始、不同入口分别负责什么”
- 首页应覆盖：
  - 白板模式：课堂讲解与发起互动
  - 互动管理：待发布任务、历史、明细、分析
  - 课堂主流程：准备内容 -> 进入白板 -> 发起互动 -> 查看结果 -> 回到互动管理复盘
  - 常用玩法：普通任务、双人 PK、单题抢答、全班挑战
- 设计要求：
  - 不做复杂后台首页
  - 优先老师端
  - 信息架构清晰，适合作为上线前的默认落地页

### P1-9 数字化教具管理
- 新增“数字化教具”独立能力，用于管理现有静态 HTML 教具
- 目标：
  - 统一管理当前约 200 个数字化教具
  - 支持批量导入
  - 支持教师在白板模式中调用
  - 预留 AI 生成 HTML 教具接入
- 第一阶段范围：
  - 后台教具列表 / 编辑 / 上下架
  - manifest + 文件目录批量导入
  - 白板模式接入教具库入口
