# 互动课堂优化路线图

## 1. 目标

下一阶段不再扩散产品边界，集中做两件事：

1. 把实时状态做稳
2. 把互动链路做成标准化产品动作

同时将入口职责固定：
- 白板模式：互动主入口
- 互动管理：任务、历史、分析入口

## 2. 第一阶段：稳定性优化

### 2.1 运行态统一
统一对象：
- 白板模式
- 互动管理
- 学生端
- 挑战状态

原则：
- 运行中状态只认服务端运行态
- 历史状态只认历史接口
- 本地状态只做 UI 辅助

验收：
- 白板、互动管理、学生端状态一致
- 不再出现已参与/已结束状态冲突

### 2.2 实时反馈优化
重点：
- 提交后立即有反馈
- 排名与结果秒级刷新
- 白板显示稳定，不做第二套业务推断

验收：
- 没有“点击后无反应”
- 没有“学生已提交但老师端没更新”

### 2.3 主链路回归
覆盖：
- 白板发题
- 普通任务
- 双人 PK
- 单题抢答
- 全班挑战
- 晚加入
- 断线重连
- 教师提前结束

## 3. 第二阶段：产品化收口

### 3.1 白板模式
白板模式只负责：
- 白板讲解
- 发起互动
- 课堂中实时查看互动过程

不再承担：
- 复杂历史管理
- 复杂分析入口

### 3.2 互动管理
互动管理只负责：
- 待发布任务
- 历史任务
- 明细查看
- 数据分析

不再承担：
- 白板讲解主流程
- 主课堂入口角色

### 3.3 发起动作标准化
统一动作：
- 发布任务
- 发起双人 PK
- 发起单题抢答
- 发起全班挑战

要求：
- 白板和互动管理的动作命名一致
- 校验提示一致
- 成功后的状态变化立即可见

### 3.4 结果与结束态标准化
统一：
- 提交反馈
- 等待反馈
- 结束态
- 结果摘要

## 4. 当前系统对应修改点

### 前端
重点区域：
- 白板实时状态
- 学生参与态
- 挑战结果态
- 白板与互动管理入口分工

### 后端
重点区域：
- 运行中 room/challenge state
- WebSocket 广播 payload
- 历史接口按场次查询

## 5. 暂不进入

当前不进入：
- 学习包回归
- 大屏过程型互动实现
- 新复杂题型扩展
- 课后约战
- 平台化后台扩张

## 6. 执行顺序

### P0
- 运行态统一
- 实时反馈稳定
- 回归链路固化

### P1
- 白板 / 互动管理分工收口
- 发起动作标准化
- 提交反馈与结束态统一

### P2
- 老师点评链路优化
- 支付联调
- 会员转化体验优化

## 7. 成功标准

如果下面 4 条成立，说明这一轮优化成功：

1. 白板、互动管理、学生端不再出现明显状态不一致
2. 老师发起与结束互动不需要靠刷新修状态
3. 学生提交后状态明确，结果同步稳定
4. 老师能清楚区分：白板负责上课，互动管理负责任务与复盘
