## 1. 模块定位

课前准备模块不再以“手动从零设计题”为核心，而是升级为：

**AI 出题工作台**

老师在上课前通过 AI 快速生成课堂任务，再在统一编辑器中完成预览、修改和发布前确认。

一句话定义：

**课前准备 = AI 生成课堂任务 + 人工确认编辑 + 进入课堂待发布**

---

## 2. 产品目标

本模块的目标是：

1. 大幅减少老师把题目录入系统的时间成本。
2. 支持从已有题目内容快速转成系统可识别的结构化题目。
3. 支持老师通过自然语言直接生成题目和答案。
4. 让 AI 输出先成为草稿，再由老师预览和修改。
5. 与课堂模式自然衔接，做到“生成后即可进入待发布队列”。

---

## 3. 创建方式

创建题目统一保留两种 AI 方式，不再把“纯手动从零创建”作为主入口。

### 3.1 方式一：导入题目内容并由 AI 识别

老师提供已有题目内容，系统通过 AI 自动分析并结构化。

支持输入：

1. 粘贴题目文本到编辑框
2. 上传 `Word(.docx)` 文件

AI 需要自动识别：

1. 题型
2. 题干
3. 选项
4. 正确答案
5. 解析
6. 建议倒计时
7. 知识点

### 3.2 方式二：输入需求由 AI 自动生成

老师输入想考察的内容，由 AI 自动生成题目和答案。

支持输入参数：

1. 主题 / 知识点
2. 题量
3. 题型
4. 难度
5. 是否带解析
6. 是否用于课堂实时发布

示例：

“围绕 would you mind 和 could you possibly 的区别，生成 5 题中等难度课堂互动题，包含单选、多选和判断。”

---

## 4. 总体流程

统一流程如下：

1. 老师进入课前准备
2. 选择 AI 创建方式
3. 导入题目内容或输入生成需求
4. AI 输出结构化题目草稿
5. 老师进入统一预览与编辑器
6. 修改题目内容、答案、展示策略
7. 完成发布检查
8. 标记为待发布
9. 进入课堂模式调用

结论：

**AI 负责生成初稿，老师负责最终确认。**

---

## 5. 与课堂模式的边界

### 5.1 课前准备负责

1. 选择 AI 创建方式
2. 上传或粘贴题目
3. 输入出题需求
4. AI 识别 / AI 生成
5. 预览题目
6. 修改题目
7. 配置随机展示规则
8. 完成发布检查
9. 标记为待发布

### 5.2 课堂模式负责

1. 查看待发布任务队列
2. 查看实时发布队列
3. 按题或按任务组发布
4. 暂停、删除、结束任务
5. 查看实时结果
6. 查看在线学生

## 6. 页面结构

课前准备模块建议保留 3 个页面：

1. `任务组列表页`
2. `AI 出题与编辑页`
3. `发布检查页`

页面流转：

`任务组列表 -> AI 创建 -> 预览与编辑 -> 发布检查 -> 课堂模式`

---

## 7. 页面需求

## 7.1 任务组列表页

目标：
让老师快速查看已有任务组并发起新的 AI 出题任务。

核心能力：

1. 查看全部任务组
2. 搜索任务组名称
3. 按状态筛选
4. 新建 AI 任务组
5. 复制任务组
6. 删除任务组
7. 归档任务组
8. 进入 AI 编辑页
9. 进入发布检查页

## 7.2 AI 出题与编辑页

这是本模块的核心页面。

页面分为 4 个区域：

1. `创建入口区`
2. `AI 结果列表区`
3. `题目预览区`
4. `题目编辑区`

### 创建入口区

需要明确展示两个入口：

1. `导入题目内容`
2. `AI 生成题目`

#### 导入题目内容

支持：

1. 文本粘贴框
2. Word 上传
3. 多题连续识别

#### AI 生成题目

支持：

1. 输入需求描述
2. 选择题量
3. 选择题型
4. 选择难度
5. 选择是否带解析

### AI 结果列表区

显示：

1. 本次生成的全部题目
2. 题型标签
3. 题干摘要
4. 识别 / 生成状态
5. 是否缺少答案
6. 是否存在风险提示

### 题目预览区

生成完成后，老师必须先看到学生端效果。

支持：

1. 学生端题目预览
2. 答题前预览
3. 结果展示预览
4. 手机 / 平板预览

### 题目编辑区

AI 生成完成后，老师可以继续修改。

支持：

1. 修改题干
2. 修改选项
3. 修改正确答案
4. 修改解析
5. 修改倒计时
6. 修改展示策略
7. 设置答案位置随机

## 8. 编辑规则

虽然创建方式改成 AI，但生成后仍使用统一编辑器。

编辑器需要支持：

1. 单选题编辑
2. 多选题编辑
3. 填空题编辑
4. 判断题编辑
5. 配对题编辑

支持的修改动作：

1. 新增题目
2. 删除题目
3. 复制题目
4. 调整顺序
5. 修改答案
6. 修改解析
7. 修改随机展示策略

---

## 9. 新增重点功能：答案位置随机

这个功能不建议做成“直接改动正确答案内容”，而应该做成：

**选项随机展示策略**

系统保存标准答案与标准选项，发布或练习时再决定是否随机打乱位置。

### 适用题型

1. 单选题
2. 多选题
3. 配对题右侧列表

### 配置方式

老师可设置：

1. 不随机
2. 每次发布随机
3. 每次学生作答随机

### 设计原则

1. 题目原始结构保持稳定
2. 正确答案存标准索引或标准 key
3. 展示层单独处理随机顺序

---

## 10. AI 输出要求

AI 输出不能是松散文本，必须是结构化题目数据。

每道题至少需要生成：

1. 题型
2. 题干
3. 选项或内容结构
4. 正确答案
5. 解析
6. 建议倒计时
7. 难度
8. 知识点

建议保留 AI 置信度字段：

1. `high`
2. `medium`
3. `low`

当置信度较低时，在编辑器中提示老师重点确认。

## 11. 数据模型调整

创建方式改为 AI 后，任务组和题目需要补充来源信息。

### 11.1 任务组

```ts
TaskGroup {
  id: string
  classId: string
  title: string
  status: 'draft' | 'ready' | 'live' | 'archived'
  source: {
    mode: 'ai_import' | 'ai_generate'
    rawInput?: string
    fileName?: string
    prompt?: string
  }
  questions: TaskQuestion[]
  settings: {
    defaultCountdownSeconds: number
    publishMode: 'manual' | 'sequence'
    showCorrectAnswerAfterEnd: boolean
    showResultsToStudents: boolean
  }
  createdAt: string
  updatedAt: string
}
```

### 11.2 题目

```ts
TaskQuestion {
  id: string
  type: 'single_choice' | 'multiple_choice' | 'fill_blank' | 'true_false' | 'matching'
  stem: string
  content: Record<string, unknown>
  answer: Record<string, unknown>
  explanation?: string
  aiMeta?: {
    confidence: 'high' | 'medium' | 'low'
    generatedBy: 'ai_import' | 'ai_generate'
  }
  settings: {
    countdownSeconds: number
    randomizeOptions?: boolean
    randomizeAnswerPosition?: boolean
    allowPartialScore?: boolean
    caseSensitive?: boolean
  }
  order: number
}
```

## 12. 发布检查页

目标：
在进入课堂前确认 AI 输出已经被老师确认过，且可安全发布。

核心能力：

1. 显示任务组摘要
2. 显示题型分布
3. 显示 AI 生成来源
4. 显示校验结果
5. 显示低置信度题目提醒
6. 允许修改默认倒计时
7. 允许统一设置随机展示策略
8. 标记为待发布
9. 跳转课堂模式

---

## 13. MVP 范围

当前直接进入开发时，建议 MVP 先做：

1. 文本粘贴识别
2. Word 上传识别
3. 自然语言生成题目
4. 支持 5 种题型
5. 生成后统一进入编辑器
6. 学生端预览
7. 发布检查
8. 答案位置随机展示配置
9. 标记待发布并进入课堂模式

---

## 14. 不纳入本轮范围

本轮先不做：

1. 图片 OCR 识别
2. PDF 复杂解析
3. 纯手工从零出题作为主入口
4. AI 自动直接发布，不经老师确认
5. 多轮复杂 AI 追问式出题助手

---

## 15. 开发指向

当前方案已明确：

1. 课前准备主入口改为 AI 创建
2. 手动设计不再作为主方案推进
3. 统一走“AI 输出 -> 预览 -> 修改 -> 发布检查 -> 课堂调用”

因此后续可以直接按以下方向开发：

1. AI 输入层
2. 结构化解析层
3. AI 生成层
4. 统一编辑器层
5. 预览层
6. 发布检查层

---

## 16. 结论

课前准备模块已经从“任务设计器”升级为“AI 出题工作台”。

当前明确采用的产品范式是：

**AI 负责生成题目草稿，老师负责预览、修改和最终发布确认。**
