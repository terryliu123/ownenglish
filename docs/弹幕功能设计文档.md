# 弹幕功能设计文档

## 一、需求概述

### 1.1 功能描述
课堂教学场景下，通过学生端发送、教师端控制、大屏端展示的弹幕互动系统，增强课堂氛围和学生的参与感。

### 1.2 三端角色

| 端 | 角色 | 功能 |
|---|---|---|
| 学生端 | 弹幕发送者 | 可发送弹幕，但不展示弹幕 |
| 教师端 | 弹幕控制者 | 控制开关、预设弹幕、手动输入、设置展示规则 |
| 大屏端 | 弹幕展示 | 仅展示弹幕，不显示来源 |

### 1.3 交互流程

```
学生端 ──────────────────────────────────────► 教师端 ─────────────► 大屏端
  │                                              │                        │
  │  发送弹幕                                     │ 审核/直接通过            │
  │ ──────────────────────────────────────────► │                        │
  │                                              │                        │
  │                                              │ 开启弹幕 + 预设/手动输入 │
  │                                              │ ───────────────────►   │
  │                                              │                        │ 展示弹幕
```

## 二、三端功能详细设计

### 2.1 学生端

#### 入口位置
底部工具栏，课堂模式下可见

#### 交互流程

```
┌─────────────────────────┐
│                         │
│     课堂内容区域          │
│                         │
│                         │
├─────────────────────────┤
│  [ 🎆 发送弹幕 ]       │  ← 底部工具栏按钮
└─────────────────────────┘

点击后展开：
┌─────────────────────────┐
│  🎆 发送弹幕            │
│  ─────────────────────  │
│  [ 输入弹幕内容...    ]  │
│  ─────────────────────  │
│     剩余 20 字          │
│                         │
│  [取消]    [发送]       │
└─────────────────────────┘
```

#### 功能规则

| 规则 | 限制 |
|------|------|
| 字数限制 | 20-50 字 |
| 发送间隔 | 10 秒/条（防刷屏） |
| 发送反馈 | 无（不知道自己发的会不会出现） |
| 违禁词 | 自动屏蔽 |

### 2.2 教师端

#### 入口位置
工具栏右侧，「弹幕」按钮

#### 主界面

```
┌──────────────────────────────────────┐
│  📝 题目  📊 互动  🎯 挑战  [🎆弹幕]   │
└──────────────────────────────────────┘

点击展开：
┌─────────────────────────┐
│  🎆 弹幕控制            │
│                         │
│  弹幕状态  [●开启]     │
│                         │
│  来源     ○全部 ○预设   │
│                         │
│  来源显示  [ ] 显示     │
│                         │
│  ── 预设弹幕库 ──────  │
│  [鼓励] [互动] [表扬]  │
│                         │
│  自定义 [输入框...]    │
│           [发送到大屏] │
│                         │
│  清屏    [一键清空]    │
└─────────────────────────┘
```

#### 设置项说明

| 设置项 | 选项 | 说明 |
|--------|------|------|
| 弹幕状态 | 开/关 | 控制大屏弹幕显示/隐藏 |
| 来源 | 全部/预设 | 全部=学生+预设都显示，预设=只显示预设 |
| 来源显示 | 勾选 | 勾选后弹幕显示「学生名：xxx」 |
| 速度 | 慢/中/快 | 慢=12秒/屏，中=8秒/屏，快=5秒/屏 |
| 密度 | 稀/中/密 | 稀=3条/屏，中=6条/屏，密=10条/屏 |
| 区域 | 全屏/下1/4/中部 | 弹幕出现区域 |

#### 预设弹幕库

| 类型 | 内容示例 |
|------|---------|
| 鼓励 | 太棒了！加油！👍 答对了！真厉害！ |
| 互动 | 准备好了吗？来看下一题！注意听讲！ |
| 表扬 | 全对！真厉害！继续保持！太优秀了！ |

#### 清屏功能
一键清空大屏所有弹幕，不影响继续发送新弹幕

### 2.3 大屏端

#### 渲染效果

```
┌────────────────────────────────────────────────────┐
│                                                    │
│                   白板教学内容                        │
│                                                    │
│                                                    │
│  ═══════════════════════════════════════════════>   │ ← 行1
│       ═══════════════════════════════════════>      │ ← 行2
│            ═══════════════════════════════════>    │ ← 行3
│  ═════════════════════════════════════════════>     │ ← 行4
│                                                    │
│                                                    │
└────────────────────────────────────────────────────┘
     ↑ 底部1/4区域，多行并行滚动
```

#### 渲染规则

| 规则 | 说明 |
|------|------|
| 行数 | 最多4行，同时滚动 |
| 位置 | 平均分布在区域高度内 |
| 滚动 | 从右向左，匀速运动 |
| 消失 | 超出左边界后淡出移除 |
| 复用 | 行空出后立即复用，显示新弹幕 |
| 来源 | 默认不显示，可配置显示 |

## 三、数据结构设计

### 3.1 数据库表

```sql
CREATE TABLE danmu_records (
    id VARCHAR(36) PRIMARY KEY,
    session_id VARCHAR(36) NOT NULL COMMENT '课堂ID',
    class_id VARCHAR(36) NOT NULL COMMENT '班级ID',
    sender_id VARCHAR(36) NOT NULL COMMENT '发送者ID',
    sender_name VARCHAR(100) NOT NULL COMMENT '发送者姓名（冗余存储）',
    content VARCHAR(200) NOT NULL COMMENT '弹幕内容',
    created_at TIMESTAMP DEFAULT NOW() COMMENT '发送时间'
);

-- 索引
CREATE INDEX idx_danmu_session ON danmu_records(session_id);
CREATE INDEX idx_danmu_created ON danmu_records(created_at);
```

### 3.2 WebSocket 消息

#### 学生 → 服务端

```typescript
// 发送弹幕
interface DanmuSend {
  type: 'danmu_send'
  content: string       // 弹幕内容
  student_id: string   // 发送者ID
}
```

#### 教师 → 服务端

```typescript
// 弹幕配置
interface DanmuConfig {
  type: 'danmu_config'
  enabled: boolean              // 开启/关闭
  showStudent: boolean         // 是否显示学生弹幕
  showSource: boolean          // 是否显示来源
  speed: 'slow' | 'medium' | 'fast'
  density: 'low' | 'medium' | 'high'
}

// 触发弹幕（预设或手动）
interface DanmuTrigger {
  type: 'danmu_trigger'
  content: string
}
```

#### 服务端 → 大屏端

```typescript
// 弹幕显示
interface DanmuDisplay {
  type: 'danmu_display'
  content: string
  row: number    // 分配到哪一行（0-3）
  showSource: boolean  // 是否显示来源
  sourceName?: string   // 来源姓名（可选）
}
```

### 3.3 API 接口

```typescript
// 存储弹幕
POST /api/v1/live/sessions/{session_id}/danmu
Request: { content: string }
Response: { id: string, success: boolean }

// 批量查询弹幕记录（用于课堂回顾）
GET /api/v1/live/sessions/{session_id}/danmu
Query: { limit?: number, offset?: number }
Response: { items: DanmuRecord[], total: number }
```

## 四、交互流程详解

### 4.1 弹幕发送流程

```
1. 学生点击「发送弹幕」
   └─ 弹出输入框

2. 学生输入内容并发送
   └─ 前端校验（字数、违禁词、发送间隔）
   └─ WebSocket 发送 danmu_send
   └─ 服务端存储到数据库
   └─ 静默成功（无明确反馈给学生）

3. 服务端根据配置决定是否广播
   └─ 如果 showStudent = false，不广播
   └─ 如果 showStudent = true，广播到所有大屏端

4. 大屏端收到消息
   └─ 分配空闲行
   └─ 开始滚动显示
```

### 4.2 教师控制流程

```
1. 教师点击「弹幕」按钮
   └─ 展开弹幕控制面板

2. 教师开启弹幕
   └─ 发送 danmu_config { enabled: true }
   └─ 大屏端开始渲染弹幕区域

3. 教师发送预设/自定义弹幕
   └─ 点击预设按钮或输入自定义内容
   └─ 发送 danmu_trigger
   └─ 服务端直接广播到所有大屏端

4. 教师关闭弹幕
   └─ 发送 danmu_config { enabled: false }
   └─ 大屏端清空所有弹幕
```

### 4.3 清屏流程

```
教师点击「一键清空」
  └─ 仅清空屏幕上的弹幕（不删除数据库记录）
  └─ 大屏端所有行重置
```

## 五、技术实现方案

### 5.1 技术选型

| 模块 | 方案 | 说明 |
|------|------|------|
| 渲染层 | DOM + CSS Animation | 轻量、兼容性好 |
| 滚动控制 | CSS @keyframes | 性能优 |
| 消息通信 | 复用现有 WebSocket | 无新增依赖 |
| 状态管理 | 课堂 session 内共享 | 复用现有架构 |

## 七、第三方违禁词 API

### 7.1 接入方案

使用阿里云内容安全 API 或腾讯云内容安全 API 进行违禁词检测。

### 7.2 违禁词检测流程

```
学生发送弹幕
    │
    ▼
前端预检（本地违禁词库，约200个常见词）
    │
    ▼
通过 → 发送 WebSocket 消息
    │
    ▼
不通过 → 提示「内容包含敏感词」
```

### 7.3 API 设计

```typescript
// 违禁词检测
POST /api/v1/danmu/check
Request: { content: string }
Response: {
  pass: boolean,
  filtered: string   // 过滤后的安全内容（可选）
}
```

### 7.4 本地违禁词库

预置本地违禁词表，覆盖常见敏感词，前端发送前先做本地检测，减少 API 调用。
