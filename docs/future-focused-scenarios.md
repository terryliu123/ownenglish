# 未来可精做场景记录

本文件记录“不是做更大的平台，而是把大产品中的某个场景做精”的方向，暂不进入开发排期，作为后续产品讨论输入。

## 场景 1：课堂即时讲解助手
- 目标：围绕老师当前白板内容，提供解释、总结、追问、检测题生成
- 适合原因：
  - 高频
  - 紧贴白板主线
  - 容易形成和通用 AI 工具的差异
- 第一版边界：
  - 教师端优先
  - 不做开放泛聊天
  - 优先基于当前白板文本与当前任务上下文

## 场景 2：课堂回顾与复盘助手
- 目标：一节课结束后自动生成课堂摘要、互动亮点、风险点、下次建议
- 适合原因：
  - 已有课堂回顾结构基础
  - 和 `LiveSession` / 课堂会话天然契合
  - 对老师的复盘价值明确

## 场景 3：课堂互动内容生成助手
- 目标：基于知识点快速生成可上课的互动内容
- 可覆盖：
  - 普通任务
  - PK / 抢答
  - 大屏互动素材
- 适合原因：
  - 直接服务核心课堂链路
  - 能和现有任务、大屏互动、数字化教具形成联动

## 场景 4：数字化教具编排助手
- 目标：帮助老师快速找到并串联合适的数字化教具
- 可覆盖：
  - 教具搜索推荐
  - 按课题推荐
  - 课堂使用顺序建议
- 适合原因：
  - 现有教具库已成型
  - 很适合做“拿来即用”的差异化体验

## 场景 5：学生端受控学习辅助
- 目标：在老师控制下为学生提供解释、提示、思路，不直接给答案
- 适合原因：
  - 对课堂有即时价值
  - 可通过开关、提示词、次数限制保持可控
- 风险：
  - 容易越界成答案助手
  - 必须强约束，不适合一开始做开放问答

## 场景 6：课堂控制助手
- 目标：帮助老师识别课堂状态并给出控场建议
- 可覆盖：
  - 谁长期未参与
  - 推荐点谁回答
  - 谁适合加入 PK
  - 哪些学生未提交
- 适合原因：
  - 对课堂管理直接有帮助
  - 可和左侧学生区后续升级联动

## 当前优先级建议
按产品价值和实现现实性，建议优先级如下：

1. 课堂即时讲解助手
2. 课堂回顾与复盘助手
3. 课堂互动内容生成助手
4. 数字化教具编排助手
5. 学生端受控学习辅助
6. 课堂控制助手

## 当前不进入范围
以下方向当前不建议进入排期：

- 通用教育大模型平台
- 泛学生课后学习产品
- 大而全内容社区
- 重型教务 / 学情管理平台
- 大规模开放聊天式 AI
