# 课堂会话与课堂回顾设计稿 v1.1

## 1. 目标

当前系统已经有白板模式、互动管理、任务发布与历史、PK/抢答/全班挑战、数字化教具和大屏互动，但缺少”**一节课**”这个上层会话模型，导致：

- 数据分析缺少课堂周期维度
- 白板、互动管理、学生端的恢复边界不清晰
- WebSocket 很难区分上一节课和本节课
- 课后无法按”整节课”回顾课堂过程

本设计补齐两层：

1. **课堂会话**（产品概念，技术实现复用并扩展现有 `LiveSession`）
2. **课堂回顾**（新增教师端一级菜单）

形成闭环：

`开始本节课 -> 上课进行中 -> 结束本节课 -> 课堂回顾`

## 技术口径说明

- **产品层**: 课堂会话 / 课堂回顾
- **技术层**: 复用并扩展现有 `LiveSession` 模型
- **统一字段名**: `live_session_id`（不引入新的 classroom_session_id）
- **改动原则**: 轻量补层，避免大重构

## 2. 产品结论

### 2.1 显式开始本节课

产品上增加明确动作：

- `开始本节课`
- `结束本节课`

不建议只靠自动触发。自动触发会把试操作、误点击和真实课堂混在一起，污染数据。

### 2.2 课堂回顾

教师端新增一级菜单：

- `课堂回顾`

它负责按“节课”展示：

- 本节课的开始结束时间
- 本节课中发起过的任务、挑战、教具、大屏互动
- 本节课学生参与情况
- 本节课的结果摘要

### 2.3 白板与互动管理职责

- **白板模式**：课堂主入口，负责开始/结束本节课和课堂现场操作（唯一开始入口）
- **互动管理**：任务和结果管理，只显示当前课堂状态，不做主开始入口
- **课堂回顾**：按节课查看整节课（第一版仅教师可见）

## 3. 数据模型

### 3.1 LiveSession（扩展现有模型）

**现有字段**：
- `id`
- `class_id`
- `group_id` - 关联的任务组
- `topic`
- `status` - 当前为 active, ended
- `started_at`
- `ended_at`

**新增字段**：
- `teacher_id` - 冗余但有用，方便查询
- `title` - 课堂标题（可选，默认为"第N节课"）
- `entry_mode` - 入口模式：whiteboard, interaction_management, bigscreen_activity
- `duration_seconds` - 课堂时长（秒），计算字段
- `summary_json` - 课堂摘要统计（JSON格式）

**状态扩展**：
- `active` - 进行中
- `ended` - 已结束
- `cancelled` - 已取消（空课堂自动取消）

**summary_json 结构**：
```json
{
  "attended_student_count": 26,
  "participated_student_count": 22,
  "task_count": 5,
  "challenge_count": 2,
  "teaching_aid_count": 3,
  "bigscreen_activity_count": 1,
  "total_duration_seconds": 2520
}
```

### 3.2 LiveSessionEvent（新增模型）

记录本节课内发生的关键事件：

- `id` - UUID
- `live_session_id` - 外键 -> live_sessions.id
- `event_type` - 事件类型（字符串）
- `payload_json` - 事件详细数据（JSON）
- `created_at` - 事件发生时间

**事件类型**：
- `session_started` - 课堂开始
- `session_ended` - 课堂结束
- `task_published` - 任务发布
- `task_ended` - 任务结束
- `challenge_started` - 挑战开始
- `challenge_ended` - 挑战结束
- `bigscreen_started` - 大屏互动开始
- `bigscreen_ended` - 大屏互动结束
- `teaching_aid_opened` - 教具打开
- `student_joined` - 学生加入
- `student_left` - 学生离开
- `hand_raised` - 举手
- `share_requested` - 分享请求
- `ai_used` - AI使用

### 3.3 运行记录挂接

**需要添加 `live_session_id` 字段的表**：

- ✅ `LiveChallengeSession` - 挑战会话（新增字段，可空）
- ✅ `BigscreenActivitySession` - 大屏互动会话（新增字段，可空）
- ✅ `TeachingAidSession` - 教具会话（新增字段，可空）

**不需要添加的表**（已有关联）：

- ❌ `LiveTask` - 已有 session_id 字段关联到 live_sessions
- ❌ `LiveTaskGroupSubmission` - 已有 session_id 字段关联到 live_sessions

## 4. WebSocket 设计影响

### 4.1 当前问题

现在大多数实时状态只按 `class_id` 聚合，缺少”这是谁的哪一节课”的边界。

### 4.2 改造方案

**房间结构不大改，只在内存房间内挂当前 live_session_id**：

- 房间仍按 `class_id` 管理：`manager.class_rooms[class_id]`
- 房间内增加字段：`live_session_id`
- 一个班同一时刻只允许 1 个 active 的 LiveSession

**必须做到**：

- `room_info` 返回 `live_session_id`
- 所有关键实时事件透传 `live_session_id`

**建议覆盖的 WebSocket 事件**：

- `room_info`
- `task_group_published`
- `task_group_ended`
- `challenge_started`
- `challenge_progress_updated`
- `challenge_scoreboard_updated`
- `challenge_ended`
- `bigscreen_started`
- `bigscreen_ended`

### 4.3 恢复规则

老师进入白板时：

1. 检查是否存在 active 的 LiveSession
2. 如果存在，优先恢复（显示”恢复本节课”）
3. 如果不存在，显示”开始本节课”
4. 不允许无提示直接再开一节新课

### 4.4 结论

从 WebSocket 角度，这不是可选增强，而是必要结构。

## 5. 课堂回顾页面

### 5.1 路由和权限

- **前端路由**: `/teacher/classroom-review`
- **后端 API**: `/api/v1/live/sessions`
- **权限**: 第一版仅教师可见，学生端暂不开放

### 5.2 列表页

每条课堂记录展示：

- 班级
- 标题
- 开始时间
- 结束时间
- 时长
- 本节课互动次数
- 参与人数
- 状态
- 快速摘要

### 5.3 详情页

建议 4 个区块：

1. **基础信息**: 班级 / 时间 / 时长 / 老师 / 入口模式
2. **课堂时间线**: 开始上课 -> 发布任务 -> PK -> 教具 -> 结束
3. **参与摘要**: 到课 / 参与 / 提交 / 举手 / 分享
4. **结果摘要**: 任务、挑战、大屏互动、教具使用

## 6. 白板模式交互

白板顶部工具条新增课堂会话控制：

- **开始本节课** - 创建新的 active LiveSession
- **恢复本节课** - 恢复已有的 active LiveSession（重连场景）
- **结束本节课** - 将 active LiveSession 标记为 ended
- **当前课堂计时** - 显示本节课已进行时长

**开始入口规则**：

- 只在白板模式提供开始/结束入口
- 互动管理只显示当前课堂状态，不做主开始入口

## 7. 空课堂处理规则

为避免无效课堂记录污染数据，增加自动取消规则：

**触发条件**（同时满足）：

1. 开始后 3 分钟内没有任何课堂动作（发布任务、挑战、教具等）
2. 没有学生进入课堂

**处理方式**：

- 自动将 status 标记为 `cancelled`
- 不计入课堂回顾的有效记录

## 9. 实施建议

不要大重构。按”轻量补层”推进：

1. 先扩展 `LiveSession` 模型，新增 `LiveSessionEvent` 模型
2. 再补开始/结束本节课的后端接口和前端交互
3. 再让运行态绑定 `live_session_id`（挑战、大屏、教具）
4. 最后补课堂回顾列表与详情页面

## 10. 验收标准

- 老师可以显式开始和结束本节课
- 一节课形成一条清晰的 LiveSession 记录
- 有独立一级菜单 `课堂回顾`（路由 `/teacher/classroom-review`）
- WebSocket 和运行态可区分不同课堂会话（通过 live_session_id）
- 数据分析获得课堂周期维度
- 空课堂自动取消，不污染数据
- 重连时能正确恢复到当前课堂会话
