# 大屏互动系统设计 v1

## 1. 文档目标

本文件定义第二阶段主线的设计范围，覆盖以下 4 个部分：

1. 大屏互动系统整体架构
2. 大屏互动内容模型与活动模型
3. 第一批大屏玩法设计
4. 教师侧内容编排工具设计

本文件只做设计，不直接改现有平板互动主链路。

## 2. 设计结论

结论先固定：

1. 大屏互动不应继续沿用当前“任务发布 -> 学生提交 -> 明细 / 分析”模型。
2. 大屏互动不应单独做一套完全独立的题库。
3. 正确结构是：
   - 共用内容素材层
   - 独立活动编排层
   - 独立活动会话层
   - 独立玩法渲染层
4. 大屏互动是第二条产品线，不替代当前平板参与型互动。
5. 当前白板 / 互动管理系统继续稳定运行；大屏互动后续以独立模块接入。

## 3. 产品定位

### 3.1 当前主线

当前主线仍然是：

- 白板模式：课堂现场与发起互动
- 互动管理：待发布任务、历史、明细、分析
- 平板参与：全员实时参与、实时提交、课堂反馈

### 3.2 第二阶段新增主线

新增主线是：

**大屏过程型互动**

它的目标不是让每个学生都提交答案，而是：

- 少量代表参与
- 全班围观
- 强过程
- 强节奏
- 强表现

### 3.3 不做的方向

大屏互动阶段明确不做：

- 把所有现有题型直接搬上大屏
- 继续沿用学生逐题提交和分析模型
- 所有人同时在大屏操作
- 与希沃式硬件课堂正面竞争

## 4. 与现有系统的关系

### 4.1 复用什么

大屏互动复用现有系统的：

- 内容素材
- 图片资源
- 题目基础结构
- 白板作为课堂主入口
- 权限、会员、支付体系

### 4.2 不复用什么

大屏互动不复用现有系统的：

- 任务组发布模型
- 学生逐条提交模型
- 查看明细 / 查看分析模型
- 当前 challenge session 的结果结构
- 平板端作答页

### 4.3 现有系统需要改什么

当前系统后续需要新增，但不能破坏现有运行：

1. 增加“大屏互动”独立模块入口
2. 增加内容素材的能力标签
3. 增加活动包与活动会话模型
4. 增加玩法渲染器框架
5. 增加教师侧活动编排工具

### 4.4 明确边界

为保证不影响现有系统，第二阶段约束固定为：

1. 不改当前平板互动协议名和主链路
2. 不把大屏活动硬塞进现有任务历史 / 明细 / 分析
3. 不让大屏活动依赖学生端登录和提交
4. 不破坏白板模式现有互动入口

## 5. 系统架构

大屏互动系统建议拆成 4 层。

### 5.1 内容素材层

职责：

- 描述题目内容本身
- 不关心是平板互动还是大屏互动

建议模型：

`ContentAsset`

核心字段：

- `id`
- `title`
- `content_type`
- `payload`
- `difficulty`
- `tags`
- `supports_device_interaction`
- `supports_bigscreen_interaction`
- `supports_competition`

首批支持的 `content_type`：

- `matching`
- `sorting`
- `classification`
- `image_reasoning`
- `error_correction`

### 5.2 活动编排层

职责：

- 定义怎么玩
- 定义回合数、计分、参与方
- 不负责具体渲染

建议模型：

`ActivityPack`

核心字段：

- `id`
- `title`
- `activity_type`
- `participant_mode`
- `content_asset_ids`
- `round_count`
- `time_limit_seconds`
- `scoring_rule`
- `win_rule`
- `status`

首批 `activity_type`：

- `duel`
- `arena`
- `timed_run`

首批 `participant_mode`：

- `student_vs_student`
- `team_vs_team`
- `challenger_vs_holder`

### 5.3 活动会话层

职责：

- 记录当前这一场活动的运行状态
- 负责回合推进、分数、胜负

建议模型：

`ActivitySession`

核心字段：

- `id`
- `class_id`
- `activity_pack_id`
- `activity_type`
- `status`
- `participant_sides`
- `current_round`
- `current_asset_id`
- `scoreboard`
- `started_at`
- `ended_at`
- `winner_side_id`

### 5.4 渲染层

职责：

- 每个游戏单独界面
- 统一外壳，不统一主画面

统一外壳负责：

- 顶部状态栏
- 双方信息
- 分数
- 回合信息
- 计时器
- 结果层
- 控制按钮

独立渲染器负责：

- `MatchingDuelRenderer`
- `SortingDuelRenderer`
- `ClassificationDuelRenderer`
- `ImageReasoningDuelRenderer`
- `ErrorCorrectionDuelRenderer`

## 6. 参与者模型

当前大屏互动不能默认绑定学生个人，因此参与者模型需要独立。

建议模型：

`ParticipantSide`

核心字段：

- `id`
- `label`
- `type`
- `member_ids`
- `display_color`
- `avatar_mode`

`type` 取值：

- `student`
- `team`
- `anonymous_side`

这允许后续支持：

- 张三 vs 李四
- A 组 vs B 组
- 擂主 vs 挑战方

## 7. 第一批玩法设计

第一批只做 3 种玩法，且只接过程型内容。

### 7.1 双人对战

目标：

- 两方围绕同一批内容做过程型竞技
- 比正确率和速度

首批接入内容：

- 配对
- 排序
- 分类归组

运行方式：

1. 教师选择活动包
2. 选择两方参与者
3. 开始活动
4. 按回合推进
5. 显示总分与胜者

结果保留：

- 参与方
- 总分
- 用时
- 胜者

不进入：

- 学生逐题明细
- 任务分析

### 7.2 擂台赛

目标：

- 当前胜者留场
- 新挑战者上场
- 形成连续守擂

首批接入内容：

- 配对
- 排序
- 分类归组

运行方式：

1. 教师设定初始擂主
2. 选择挑战者
3. 进行一轮活动
4. 胜者继续留场
5. 记录连胜

结果保留：

- 当前擂主
- 连胜次数
- 最近若干轮结果

### 7.3 限时闯关

目标：

- 在限定时间内连续完成若干回合
- 看完成数和总分

首批接入内容：

- 配对
- 排序
- 分类归组

运行方式：

1. 教师选择活动包
2. 选择参与者
3. 设置闯关时间
4. 系统连续下发内容
5. 时间结束后结算

结果保留：

- 完成关卡数
- 总分
- 总用时

## 8. 第一批内容范围

第一批建议只做 3 类内容，控制复杂度。

### 8.1 配对

适合大屏的原因：

- 过程清晰
- 对战感强
- 全班能看懂

### 8.2 排序

适合大屏的原因：

- 拖拽和重排过程明显
- 节奏适合课堂

### 8.3 分类归组

适合大屏的原因：

- 视觉上容易理解
- 兼顾正确率和速度

第一批不建议接入：

- 简单单选
- 简单判断
- 长阅读
- 口语
- 开放题

## 9. 教师侧内容编排工具

教师端需要新加一个独立编排入口，不放在当前“待发布任务”里。

建议模块名称：

`大屏互动`

### 9.1 活动包管理页

功能：

- 查看活动包列表
- 新建活动包
- 编辑活动包
- 复制活动包
- 启停活动包

### 9.2 活动包编辑器

功能：

- 选择玩法类型
- 选择参与模式
- 选择内容素材
- 配置回合数
- 配置计时规则
- 配置计分规则

### 9.3 内容素材选择器

功能：

- 按内容类型筛选
- 按标签筛选
- 预览素材
- 批量加入活动包

### 9.4 课堂启动器

教师在白板模式中增加：

- 打开大屏互动
- 选择活动包
- 选择参与方
- 开始活动

白板仍是课堂入口，但大屏活动的配置和启动逻辑独立于当前任务发布流。

## 10. 对现有系统的修改清单

第二阶段实际开发时，建议只改这些模块：

### 10.1 新增

新增前端模块：

- `client/src/features/bigscreen-activities/`
- `client/src/pages/teacher/BigscreenActivities.tsx`
- `client/src/pages/teacher/BigscreenActivityRun.tsx`

新增后端模块：

- `server/app/api/v1/bigscreen_activities.py`
- `server/app/models/activity_pack.py`
- `server/app/models/activity_session.py`

### 10.2 现有模块只做轻改

现有模块只允许轻改：

- 白板模式增加入口按钮
- 会员权益增加“大屏互动”开关或版本策略
- 内容素材增加能力标签

### 10.3 不应该修改的部分

以下部分不作为第二阶段改造对象：

- 当前平板互动的 challenge session 模型
- 当前任务历史 / 明细 / 分析主流程
- 当前学生端提交协议
- 当前白板实时互动主链路

## 11. 实施顺序

建议按这个顺序推进：

1. 定稿内容素材层与活动模型
2. 定稿活动包编辑器
3. 定稿双人对战运行界面框架
4. 实现配对对战
5. 实现排序对战
6. 实现分类归组对战
7. 实现擂台赛
8. 实现限时闯关

## 12. 验收标准

第二阶段设计验收标准：

1. 不影响当前平板互动系统运行
2. 不影响当前白板 / 互动管理职责
3. 大屏互动拥有独立内容模型、活动模型、会话模型、渲染层
4. 第一批玩法范围和内容范围明确，不再摇摆
5. 教师侧编排和启动路径明确

## 13. 一句话结论

第二阶段不再是“给现有任务体系继续打补丁”，而是：

**在复用内容素材的前提下，独立建立一套大屏过程型互动系统。**
